Thinking 0.6

使用重定向书写如下,此为command文件

result文件

echo Shell Start直接将Shell Start作为字符串输出

echo echo Shell Start``将反引号中的命令输出作为输出内容，即输出Shell Start

二者输出内容无区别

echo echo $c > fileShell解析第一个echo并将之后的内容全部作为字符串输出。

echo echo $c > file``由于反引号的存在，Shell先解析反引号中的内容，即echo $c > file输出为$c > file所以命令行输出应为$c > file

实验难点

exercise方面

笔者在0.4卡了一会，主要原因有两个，一个是csc目录下的Makefile递归调用的问题，这个可以用$(MAKE) -c 目录名来实现，（不过我比较懒，在csc的Makefile中利用-o直接指定了文件生成路径，其实也不失为一种办法）

另一个，笔者在编译时，一直报错，提示我找不到include的文件，找了好久，才发现main.c的依赖不是标准库，而是放在include目录下，这个时候，就要使用gcc的-I选项，指定头文件路径来实现

Exam方面

没什么说的，Lab0的exam提示给的很全，跟着做很难不会^^

Extra方面

Extra主要考察.sh脚本的实现方法，笔者得了70分，主要原因还是对题目理解不深，可能是因为考试加的debuff吧，另，Extra给出了很多关于文件和目录的权限问题，笔者总是忘记改权限，导致苦等3分钟

Extra最后三个测试点是关于uniq，sort和awk的
